# Engineering Definition of Done

## Purpose

A work item is done only when the implementation, evidence, tests, documentation, and validation impact are complete enough for accountable review.

## Done Criteria

A work item is done only when it includes, where applicable:

- Implementation.
- Deterministic behavior.
- Normalized findings, signals, claims, gaps, and evidence.
- Tests.
- Sample fixtures.
- Rule updates.
- Profile updates.
- Evidence output updates.
- Report updates.
- Documentation updates.
- Changelog update.
- Known limitations.
- Validation impact note.
- No regression gate failure.

## Evidence Criteria

Material findings must be traceable to evidence. Evidence output changes must be reflected in manifest generation and documentation where applicable.

## Quality Criteria

The work must preserve:

- Local-first execution.
- Offline capability.
- Non-AI operation.
- No telemetry.
- Conservative defaults.
- English-only artifacts.
- Analyzer status transparency.
- Baseline rule governance.

## Completion Evidence

Completion should record:

- Files changed.
- Tests run.
- Validation commands run.
- Sample scan result when applicable.
- Known limitations.
- Release or follow-up recommendation.
