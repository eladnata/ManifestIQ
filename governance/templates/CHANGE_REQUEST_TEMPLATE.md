# Change Request

## Change ID

`CR-YYYY-NNN`

## Title

Short descriptive title.

## Change Type

Allowed values:

- analyzer
- rule
- profile
- scoring
- decision_logic
- report
- evidence_schema
- validation
- governance
- cli
- refactor
- bug_fix
- security_fix
- documentation

## Reason

Why this change is needed.

## Scope

What will change.

## Out of Scope

What will not change.

## Affected Modules

List affected code modules.

## Affected Rules

List affected baseline or custom rules.

## Affected Profiles

List affected profiles.

## Affected Evidence Outputs

List expected evidence output changes.

## Affected Reports

List expected report changes.

## Risk Level

Allowed values: High, Medium, Low.

## Validation Required

Yes or No.

## Tests Required

List required tests and fixtures.

## Documentation Required

List required documentation updates.

## ADR Required

Yes or No. If yes, link the ADR.

## Rollback Plan

How to revert or mitigate the change.

## Approval Role

Maintainer, AppSec, CISO, CTO, Architecture, SOX, ITGC, Legal, Privacy, or Release Manager.

## Known Limitations

Known limits, assumptions, or deferred work.
