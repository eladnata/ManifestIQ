# Release Checklist

## Release ID

`REL-YYYY-NNN`

## Candidate Version

Scanner version and ruleset version.

## Checklist

- [ ] Tests passed
- [ ] Sample scan passed
- [ ] Evidence package generated
- [ ] Manifest complete
- [ ] Adversarial validation passed
- [ ] Gold set validation completed where available
- [ ] Portfolio validation completed where available
- [ ] Trend gate passed or warning accepted
- [ ] Rule governance checked
- [ ] Baseline rules protected
- [ ] Skipped analyzers explicitly documented
- [ ] Known limitations updated
- [ ] Changelog updated
- [ ] ADRs updated if needed
- [ ] Documentation updated
- [ ] Release decision recorded

## Release Decision

Allowed values:

- Blocked
- Conditional
- Approved for internal validation
- Approved for pilot
- Approved for production use

## Approver Role

Role accountable for the decision.

## Accepted Warnings

List warnings, approver role, reason, and expiry.

## Known Limitations

Known limits carried into this release.
